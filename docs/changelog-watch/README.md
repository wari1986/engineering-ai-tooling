# Changelog watch loop

## Purpose

Track changes to AI tooling policy, Codex rules, MCP catalog, and reusable workflows.

## Cadence

- Weekly review by `AI_GOVERNANCE_OWNER`.
- Monthly summary to stakeholders.

## Signals to monitor

- Changes under `configs/codex/`.
- Changes under `mcp/`.
- Changes to reusable workflows.

## Notifications

- Email: `INCIDENT_EMAIL`
- Slack: `SLACK_INCIDENT_CHANNEL`

## Reporting format

- Change summary
- Risk assessment
- Required action items
