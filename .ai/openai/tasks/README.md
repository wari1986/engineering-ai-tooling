# Task Briefs

## When to create a task brief

Create a task brief for work that is:

- Multi-session.
- Ambiguous or exploratory.
- Cross-module or cross-repository.

## How to use briefs

- Start from `_template.md`.
- Update status, decisions, and validation after each session.
- Link related PRs and issues.

## Naming convention

`YYYY-MM-DD-short-title.md`
