# Claude Code Instructions

Use this repository as the source of truth for AI governance, security, and engineering standards.

## Core references

- Governance and security policies: `docs/`
- Engineering standards (taste layer): `docs/engineering-standards/`
- MCP catalog and policies: `mcp/`
- Claude configuration pack: `configs/claude/`

## Working style

- Follow the governance and security guidance before proposing automation.
- Keep changes minimal and auditable; prefer updating templates over inventing new structures.
- Use placeholders (e.g., `ORG_NAME`, `GITHUB_ORG`, `SECURITY_OWNER`) until an org rollout.
